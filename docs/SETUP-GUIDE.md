# Kason Marketing OS V3.0 — 飞书配置手把手指南

> 本指南假设你已有一个飞书企业账号。**两种部署方式可选**：

| 方式 | 耗时 | 说明 |
|------|------|------|
| 🚀 **一键自动部署** | 5分钟手动 + 5分钟自动 | Phase 1+2自动完成，推荐！ |
| ✋ **完全手动配置** | 2-3小时 | 每个步骤手动操作 |

---

## 🚀 一键自动部署（推荐）

### 前置条件

1. 在飞书开发者后台创建应用 → 获取 App ID 和 App Secret (5分钟)
2. 配置 `.env` 文件

### 执行

```bash
# Windows 用户: 双击运行
scripts/deploy.bat

# 或命令行
node scripts/auto-deploy.js
```

**自动完成的内容**:
- ✅ 创建 `Kason Marketing OS` 多维表格
- ✅ 创建 7 个数据表 (campaigns/content/leads/tasks/analytics/skills_registry/workflow_log)
- ✅ 添加 120+ 个字段（含单选/多选选项）
- ✅ 注册 47 个技能到 skills_registry 表
- ✅ 生成 `.deploy-result.json` (包含 URL 和 Token)

**仍需手动完成**:
- 飞书应用创建 (5分钟)
- 表中关联列 (跨表关联需要手动添加，约5分钟)
- 飞书工作流配置 (6个，约30分钟)
- API 服务部署 (10分钟)

## Phase 1：创建飞书应用和机器人 (20分钟)

### 步骤 1.1：创建企业自建应用

1. 打开 **飞书开发者后台** → 点击「创建企业自建应用」
2. 填写：
   - **应用名称**: `Kason Marketing OS`
   - **应用描述**: `全栈营销操作系统 - AI驱动的营销自动化平台`
   - **应用图标**: 可先跳过
3. 点击「创建」

### 步骤 1.2：获取应用凭证

创建后，左侧菜单 → 「凭证与基础信息」：

记下这两个值（之后要填入 `.env`）：
```
App ID:     cli_xxxxxxxxxxxx
App Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 步骤 1.3：添加应用能力

左侧菜单 → 「应用能力」：

| 能力 | 操作 | 用途 |
|------|------|------|
| 🤖 **机器人** | 点击「添加」 | 让用户通过 @机器人 使用技能 |
| 📊 **多维表格** | 点击「添加」 | 创建和管理营销数据表 |
| ⚙️ **工作流** | 确保已开启 | 自动化营销流程 |

### 步骤 1.4：配置机器人

添加机器人能力后：
1. 点击「机器人」进入配置页
2. **消息卡片回调**: 先留空，Phase 5 再配置
3. **事件订阅**: 先不配置
4. 点击「发布」保存

### 步骤 1.5：配置权限

左侧菜单 → 「权限管理」→ 搜索并开启以下权限：

**消息与群组**:
- [x] `im:message` — 获取与发送单聊、群组消息
- [x] `im:message:send_as_bot` — 以机器人身份发送消息

**多维表格**:
- [x] `bitable:app` — 多维表格读写权限
- [x] `bitable:app:manage` — 多维多表格管理权限

**通讯录**:
- [x] `contact:user.id:readonly` — 获取用户 ID

**工作流**:
- [x] `approval:instance` — 审批实例读写权限

### 步骤 1.6：发布应用

1. 左侧菜单 → 「版本管理与发布」
2. 点击「创建版本」
3. 填写版本号: `1.0.0`，说明: `Kason Marketing OS V3.0 初始化`
4. 点击「保存」→「申请发布」
5. 管理员审核通过（如果你是管理员，可以立即通过）

### 步骤 1.7：安装到工作台

1. 审核通过后，在飞书客户端中
2. 打开「工作台」→ 右上角「···」→「编辑工作台」
3. 搜索 `Kason Marketing OS` → 添加到工作台

### ⚙️ Workflow 1：内容发布自动化

**触发**：记录变更 → content 表 → `status` 字段 → 变为「已通过」

**步骤逐步添加**：

```
步骤1: 获取记录
  → 选择 content 表 → 记录 = 当前触发记录

步骤2: 条件判断
  → 如果 WP文章ID 为空 → 执行步骤3-6
    如果 WP文章ID 不为空 → 执行步骤7 (更新)

步骤3: 发送 HTTP 请求 (调用 WordPress API)
  → 方法: POST
  → URL: (Phase 5 部署后填入) https://你的服务器/api/content/publish
  → Body (JSON):
    {
      "title": "{{步骤1.标题}}",
      "status": "publish",
      "meta_title": "{{步骤1.SEO标题}}",
      "meta_description": "{{步骤1.SEO描述}}"
    }

步骤4: 更新 content 记录
  → 状态 = "已发布"
  → WP文章ID = {{步骤3.response.data.id}}
  → WP链接 = {{步骤3.response.data.link}}
```
