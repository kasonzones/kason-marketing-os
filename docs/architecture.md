## 数据流

### 内容营销流
```
飞书 Skill 触发 → AI 生成内容 → 多维表存储草稿
  → 飞书审批 Workflow → 审核通过
  → WordPress API 发布 → SEO 自动优化 → 多维表更新状态
```

### 线索管理流
```
网站表单提交 → Webhook 接收 → 多维表创建线索
  → 线索评分 Workflow → 自动分配销售
  → 邮件序列触发 → 互动行为回写 → 评分更新
```

### 数据分析流
```
定时触发 (每日/每周) → GA4/各平台拉取数据
  → 数据清洗聚合 → 多维表写入
  → Dashboard 刷新 → 异常告警推送
```

## 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| API网关 | Express.js (Node.js) | 轻量、生态丰富、飞书 SDK 支持好 |
| 数据存储 | 飞书多维表 | 与飞书深度集成、权限管理、实时协作 |
| 任务调度 | 飞书 Workflow + node-cron | 飞书原生 + 服务端兜底 |
| 内容发布 | WordPress REST API | WordPress 市场占有率高、API完善 |
| AI 引擎 | 多模态大模型 | 文本生成、图片生成、数据分析 |
| 部署 | 云函数 (SCF) | 免运维、按量付费、飞书生态集成 |

## 安全设计

1. **认证**: 飞书 OAuth 2.0 + API Key 双重认证
2. **传输**: HTTPS 加密传输
3. **存储**: 敏感信息使用环境变量，不硬编码
4. **权限**: 飞书权限分级 (管理员/编辑者/只读)
5. **审计**: 所有操作记录到 workflow_log 表
6. **限流**: API 网关限流 (100 req/min per tenant)
