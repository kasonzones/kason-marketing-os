# /编辑文案 (edit-copy)

## 功能描述

七步编辑法深度审查营销文案。从清晰度、语调、价值连接、证明力、具体性、情感高度到风险消除，逐维度诊断并输出优化版本。附带多角色审查面板，模拟CEO、客户、设计师、法务四个视角的交叉审阅。

本技能是文案质量的最后一道关卡——发布前用它过一遍，确保每个字都在为转化服务。

---

## 编辑七步法

```
清晰度 → 语调/声音 → 价值连接 → 提供证明 → 具体性 → 情感高度 → 消除风险
 Clarity   Voice     Value       Proof     Specificity  Emotion    Risk
```

### 第一步：清晰度审查 (Clarity)

检查文案是否"一眼就懂"：

- 标题是否在 3 秒内传达核心信息？
- 是否存在行业黑话、内部术语或模糊表达？
- 句子结构是否过长、嵌套过多？
- 信息层级是否清晰（标题 > 副标题 > 正文）？

```
🔍 清晰度审查
  ❌ 原文: "本产品采用先进的流体动力学算法实现水资源的高效利用"
  ⚠️ 问题: "流体动力学算法" — 受众不理解，信息丢失
  ✅ 建议: "智能水循环系统，每洗一辆车省 120 升水"
  📊 清晰度评分: 前 3/10 → 后 8/10
```

### 第二步：语调和声音审查 (Voice & Tone)

检查文案是否有人格、有温度：

- 读起来像人在说话还是机器在播报？
- 语调是否与品牌人设一致？
- 是否存在"假大空"套话（"卓越品质"、"行业领先"）？
- 目标受众是否感受到"你在对他们说话"？

```
🔊 语调审查
  ❌ 原文: "AutoWash Pro X1 是行业领先的全自动洗车解决方案提供商"
  ⚠️ 问题: 第三人称 + 空洞形容词 = 像新闻稿，不像对话
  ✅ 建议: "我们做了件疯狂的事：让洗车机自己赚钱，老板只管收钱"
  📊 语调评分: 前 2/10 → 后 8/10
```

### 第三步：价值连接审查 (Value Connection)

检查好处是否与受众真实需求对齐：

- 每个功能点是否都对应一个"这对我意味着什么"的好处？
- 好处是否具体、可感知、可量化？
- 是否存在"功能罗列病"（只说有什么，不说为什么重要）？
- 价值主张是否与受众的核心痛点精准匹配？

```
💎 价值连接审查
  ❌ 原文: "配备 12 组高压喷头和 5 组风干系统"
  ⚠️ 问题: 纯功能罗列，受众不知道这意味什么
  ✅ 建议: "12 组喷头 = 无死角覆盖，轮毂缝隙都洗到。5 组风机 = 出来就是干的，不用再擦"
  📊 价值评分: 前 3/10 → 后 9/10
```

### 第四步：提供证明 (Provide Proof)

检查声称是否有据可查：

- 每一个声称是否有对应的证明？
- 证明类型是否合适（数据/证言/认证/演示/案例）？
- 数字是否有来源？
- 案例是否具体（有名有姓有数据）？

```
📊 证明力审查
  ❌ 原文: "深受广大客户好评"
  ⚠️ 问题: 空洞的社会证明，零信任建立
  ✅ 建议: "山东中石化 47 站：'装了 8 个月，洗车营收 73 万，设备零故障'"
  📊 证明评分: 前 1/10 → 后 8/10
```

### 第五步：具体性审查 (Specificity)

检查文案是否具体到"可以画出来"：

- 形容词是否都可以被更具体的描述替代？
- 数字是精确的还是模糊的？
- 场景描述是否让人产生画面感？
- 是否存在"万能文案"（换一个产品名照样用）？

```
🔎 具体性审查
  ❌ 原文: "高效快速的洗车体验"
  ⚠️ 问题: 适用于任何洗车机，毫无辨识度
  ✅ 建议: "从进场到出场 3 分 12 秒，比你自己洗快 10 倍，比隔壁人工洗快 3 倍"
  📊 具体性评分: 前 1/10 → 后 9/10
```

### 第六步：情感高度审查 (Emotion)

检查文案是否从"信息传递"升级到"情感共鸣"：

- 是否触发了某种情感（紧迫感/归属感/恐惧/渴望/骄傲）？
- 是否使用了故事、类比或画面？
- 受众读完是否产生"这就是我需要的"的感觉？
- 情绪强度是否与转化目标匹配？

```
❤️ 情感审查
  ❌ 原文: "选择 AutoWash，提升您的洗车业务效率"
  ⚠️ 问题: 平淡如水，零情感驱动力
  ✅ 建议: "想象一下：周六早上，你还在被窝里，机器已经替你赚了 3,000 块"
  📊 情感评分: 前 1/10 → 后 8/10
```

### 第七步：风险消除审查 (Risk Removal)

检查是否消除了受众的所有顾虑：

- 是否存在未解答的隐含反对意见？
- 是否有消除风险的保障措施（退款/试用/保修）？
- 是否有明确的使用门槛说明？
- CTA 是否存在决策压力（太贵/太复杂/不适合我）？

```
🛡️ 风险审查
  ⚠️ 未解答的顾虑:
  ├─ "万一机器坏了怎么办？" → 补充: 48小时上门 + 备用机政策
  ├─ "我场地够不够？" → 补充: 最小仅需 4m×8m，免费场地评估
  └─ "回不了本怎么办？" → 补充: 我们敢保证：12个月不回本，差额我们补
  📊 风险评分: 前 4/10 → 后 9/10
```

## 执行流程

### 第一步：文本预处理

- 识别文案的载体和目标（来自 `context`）
- 标记关键信息点（价格/功能/承诺/CTA）
- 建立受众共情画像

### 第二步：逐维度诊断

按照七步法的固定顺序，每步产出：
1. 当前评分 (1-10)
2. 发现的问题（引用原文，标注位置）
3. 具体的优化建议（附带改写示例）
4. 优化后的评分

### 第三步：多角色审查面板

模拟四个角色交叉审阅：

```
👥 多角色审查面板
┌──────────────────────────────────────────────────────────────┐
│ 👔 CEO 视角                                                  │
│ "看完前三屏，我只记住'省水'。差异化在哪？为什么选我们不是     │
│  竞品？品牌主张太弱。"                                        │
│ 评分: 5/10                                                   │
├──────────────────────────────────────────────────────────────┤
│ 👤 客户视角 (加油站老板 老王)                                 │
│ "说了这么多高级功能，我就想问：多久回本？坏了谁来修？        │
│  有没有跟我一样的人用过？"                                    │
│ 评分: 4/10                                                   │
├──────────────────────────────────────────────────────────────┤
│ 🎨 设计师视角                                                │
│ "这段文案太密了，移动端看就是一堵墙。建议拆成3个卡片，        │
│  每个不超过两行。"                                           │
│ 评分: 6/10                                                   │
├──────────────────────────────────────────────────────────────┤
│ ⚖️ 法务视角                                                 │
│ "'最好'属于绝对化用语，有广告法风险。'保证回本'需要标注      │
│  条件和免责。"                                               │
│ 评分: 7/10                                                   │
└──────────────────────────────────────────────────────────────┘
```

### 第四步：生成优化版本

整合所有诊断结果，输出编辑后的完整文案。根据 `intensity` 参数控制改动幅度：

- **light**: 只修正明显问题（错别字、不通顺、明显歧义）
- **standard**: 逐维度优化，标注修改原因
- **heavy**: 深度重写，可能改变结构和信息架构
