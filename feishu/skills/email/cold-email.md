# /冷邮件 (cold-email)

B2B 冷邮件外展技能，专注于高回复率的精准商务开发邮件。核心原则：像同行交流而非供应商推销，每句话都必须物有所值，个性化必须与实际痛点挂钩。

## 触发词

`/冷邮件` `/cold-email`

## 核心原则

1. **同行语气，而非供应商姿态**：用 "我注意到贵公司..." 而非 "我们的产品可以..."。建立平等对话关系。
2. **每句话必须物有所值**：删除所有空洞的形容词和套话。如果一句话删掉不影响理解，就删掉它。
3. **个性化绑定痛点**：基于目标公司的公开信息（融资、招聘、新闻、技术栈）推导真实业务问题，再关联你的解决方案。

## 邮件框架

### 1. 问题导向框架 (Problem-First)
适用于：已知目标客户存在明确痛点的场景。

```
[具体观察/洞察]
↓
[该问题带来的业务影响]
↓
[简要解决方案引入]
↓
[低摩擦 CTA（非销售型）]
```

### 2. 价值证明框架 (Proof-First)
适用于：有过硬案例数据支撑的场景。

```
[一个令人印象深刻的结果]
↓
[简短说明如何实现]
↓
[你的产品/服务与此结果的关联]
↓
[开放式 CTA]
```

### 3. 触发事件框架 (Trigger-Event)
适用于：基于目标公司近期事件（融资、扩张、人事变动）的时机型外展。

```
[提及近期触发事件]
↓
[该事件通常意味着某个需求]
↓
[你如何帮助他们抓住这个机会]
↓
[时间敏感型 CTA]
```

## 投递策略

- **发送窗口**：周二至周四，当地时间 6:00-8:00 或 14:00-15:00
- **发送量**：每天 20-50 封（新域名逐步预热）
- **发送工具**：Lemlist、Instantly、Smartlead
- **追踪参数**：打开率（基准 40-60%）、回复率（基准 3-8%）、退信率（<3%）
- **预热周期**：新邮箱域名预热 2-3 周，日增 5-10 封

## 反垃圾邮件清单

- [ ] 主题行无全大写、无过多感叹号
- [ ] 正文文字/图片比例 > 80/20
- [ ] 无短链接（使用完整 URL）
- [ ] SPF、DKIM、DMARC 已验证
- [ ] 退订链接清晰可见
- [ ] 自定义追踪域已设置
