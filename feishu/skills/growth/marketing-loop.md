# /营销循环 (Marketing Loop)

> 🔁 创建周期性营销工作流，让增长自动化持续运转

## 触发词

`/营销循环` `/marketing-loop`

## 功能概述

设计并执行可复用的周期性营销回环，让每一次营销活动形成自驱动的增长飞轮。基于 9 步回环结构构建，预置 43 个经过验证的回环模板。

## 9 步回环结构

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | 🔍 回环检查 | Cycle Check — 确认当前循环状态，防止重复执行 |
| 2 | 🎯 触发器 | Trigger — 定义激活回环的条件和时机 |
| 3 | 🥅 目标 | Goal — 明确本次循环的核心 KPI |
| 4 | 🛠️ 技能调用 | Skills Used — 列出本次循环依赖的技能模块 |
| 5 | 🔄 回环主体 | Loop Body — 核心执行逻辑和行为序列 |
| 6 | ✅ 自查 | Self-Check — 执行质量校验与数据核验 |
| 7 | 📊 状态/幂等 | State/Idempotency — 确保重复触发不产生脏数据 |
| 8 | 🛑 停止条件 | Stop Condition — 定义循环终止的边界条件 |
| 9 | 📤 输出 | Output — 循环产出物（报表、触发下一个循环等） |

## AutoWash Pro X1 示例

### 示例 1：内容冷启动循环

```yaml
回环名称: AutoWash Pro X1 博客冷启动
触发器: 新博客文章发布 (Webhook → Feishu)
目标: 单篇文章 30 天内获得 1000+ 自然流量
技能调用:
  - /营销循环 (自身)
  - /SEO关键词 (keyword research)
  - /内容分发 (content distribution)
回环主体:
  第1天: 发布文章，提交 Google Search Console 索引
  第1天: 发布到 Twitter/X、LinkedIn、Facebook
  第2天: 同步到 Medium、Dev.to（canonical 链接回主站）
  第3天: Reddit 相关子版块发布讨论帖
  第7天: 检查排名，优化标题/元描述
  第14天: 更新文章内容，添加新数据/案例
  第30天: 生成流量报告，判断是否重启循环
自查:
  - 所有社交链接是否可正常访问
  - Medium/Dev.to 是否正确设置 canonical
  - GSC 索引状态是否正常
状态管理:
  - 文章 ID 作为幂等键
  - 记录每次分发渠道和时间戳
停止条件: 达到 1000 流量 OR 满 30 天
输出:
  - 📊 30 天流量报告
  - 📝 内容优化建议
  - 🔄 触发下一轮内容循环（如 ROI 为正）
```

### 示例 2：竞品监控循环

```yaml
回环名称: 全自动洗车机竞品监控
触发器: 每周一 09:00 AM (Feishu 定时工作流)
目标: 追踪 Top 5 竞品动态，发现市场机会
技能调用:
  - /竞品分析 (competitor analysis)
  - /社交媒体监控 (social listening)
  - /价格监测 (price monitoring)
回环主体:
  步骤1: 抓取竞品官网更新（产品页、定价页、案例页）
  步骤2: 监测竞品社交媒体动态（Twitter/LinkedIn/Facebook）
  步骤3: 扫描行业新闻/PR 通稿
  步骤4: 检查竞品评价平台新增评论（G2/Capterra）
  步骤5: 汇总变化点，生成差异分析
自查:
  - 数据抓取是否完整
  - 时间戳是否准确
  - 是否有漏掉的竞品
状态管理:
  - week_number + year 作为幂等键
  - 每次循环结果存入竞品数据库
停止条件: 正常完成本周监控
输出:
  - 📋 竞品动态周报（PDF + Feishu 文档）
  - 🚨 预警信号（如有重大变化）
  - 💡 应对建议列表
```

## 与 Feishu 定时工作流集成

```
触发方式:
  ┌──────────────────────────────────────┐
  │  Feishu 定时触发器                      │
  │  ├── Cron 表达式: 0 9 * * 1 (每周一早9点) │
  │  ├── Webhook URL: 外部系统回调           │
  │  └── 人工触发: /营销循环 [回环名]          │
  └──────────────────────────────────────┘
                     │
                     ▼
  ┌──────────────────────────────────────┐
  │  回环引擎                               │
  │  ├── 检查状态 → 执行主体 → 记录输出       │
  │  └── 异常处理 → 重试 → 告警通知           │
  └──────────────────────────────────────┘
                     │
                     ▼
  ┌──────────────────────────────────────┐
  │  输出分发                               │
  │  ├── Feishu 群消息通知                   │
  │  ├── Feishu 文档自动更新                  │
  │  ├── 触发下游回环 (链式调用)               │
  │  └── 数据写入 BI 看板                      │
  └──────────────────────────────────────┘
```

---

> 💡 **提示**: 每次创建新回环时，必须明确"停止条件"——没有终点的循环就是资源的黑洞。好的营销循环像心脏：收缩-舒张-收缩，有节奏、可预测、可观测。
