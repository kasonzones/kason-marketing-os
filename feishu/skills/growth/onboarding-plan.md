# /用户引导 (Onboarding Plan)

> 🚀 设计注册后的首次体验，让新用户快速到达"Aha Moment"，实现从注册到活跃的质变

## 触发词

`/用户引导` `/onboarding-plan`

## 功能概述

构建从注册完成到用户激活的完整引导体系。定义激活指标（Aha Moment），设计引导清单、空状态提示和多渠道触达策略，最大化新用户的首月留存率。

## 核心理念

```
注册 ──────────────────────────────────────────► 激活 (Aha Moment)
       │                                              │
       │   ⏱️ Time-to-Value (TTV / 价值实现时间)        │
       │   目标: 越短越好                                 │
       │                                              │
       └── 引导路径 (Onboarding Path) ──────────────────┘
            ├── 产品内引导 (In-app)
            ├── 邮件序列 (Email Sequence)
            ├── 应用内消息 (In-app Messages)
            └── 人工介入 (Human Touch — 大客户专属)
```

## 激活指标定义（"Aha Moment"）

### 如何找到你的 Aha Moment

```yaml
方法1 - 留存相关性分析:
  步骤:
    1. 列出所有关键行为事件
    2. 计算每个事件的完成率
    3. 统计"完成事件 → Day 30 留存"的相关系数
    4. 找到相关系数最高的事件 → 候选 Aha Moment
  示例:
    - Facebook: 10 天内添加 7 个好友
    - Twitter: 关注 30 个人
    - Slack: 发送 2000 条消息
    - Dropbox: 上传 1 个文件到文件夹

方法2 - 用户访谈反推:
  问题: "你第一次觉得'这东西真有用'是什么时候？"
  统计高频答案，找到共性场景

方法3 - 竞品推断:
  分析竞品的新手引导流程
  他们在引导用户完成什么行为？
```

### 常见产品的激活指标

| 产品类型 | 激活指标示例 | 目标 TTV |
|---------|-------------|----------|
| 📊 数据分析工具 | 连接数据源 + 看到第一个仪表盘 | < 5 分钟 |
| 📧 邮件营销 | 导入列表 + 发送第一封邮件 | < 15 分钟 |
| 💳 支付工具 | 完成第一笔交易 | < 3 分钟 |
| 🏭 IoT/硬件 | 设备上线 + 首次成功运行 | < 30 分钟 |
| 📝 项目管理 | 创建项目 + 邀请 1 位成员 + 创建第一个任务 | < 10 分钟 |
| 🎨 设计工具 | 完成第一个设计 + 导出 | < 10 分钟 |
| 🔧 开发者工具 | API 调用成功 + 返回正确结果 | < 5 分钟 |

## 引导模式

### 1. 清单模式 (Checklist Pattern)

```
┌────────────────────────────────────────┐
│  ✅ 创建第一个项目          [已完成]     │
│  ⬜ 邀请团队成员                          │
│  ⬜ 完成第一个任务                        │
│  ⬜ 查看分析报告                          │
│  ─────────────────────────────          │
│  📊 完成度: 25%  剩余 3 步               │
└────────────────────────────────────────┘
```

**最佳实践:**
- 3-7 步为佳，超过 7 步会吓跑用户
- 每完成一步立即显示进度变化
- 完成后给予奖励（徽章/功能解锁/祝贺动画）
- 支持跳过（不要强制）

### 2. 空状态模式 (Empty States)

```yaml
空状态设计原则:
  - 不要只显示"暂无数据"——这是浪费空间
  - 用空状态推动第一个关键行为
  - 展示最终状态的预览（让用户期待"填满"的感觉）

示例:
  ❌ 糟糕: "你还没有任何项目。" [空白页面]
  ✅ 优秀:
     🎨 插图展示一个有内容的项目页面
     💬 "创建你的第一个项目，只需 30 秒"
     🎯 [创建项目] CTA 按钮
     📖 "查看示例项目" 链接
```

### 3. 渐进式引导 (Progressive Disclosure)

```
新手模式 ──► 中级模式 ──► 高级模式
(简化版)     (标准版)     (完整版)

规则: 根据用户行为自动升级界面复杂度
```

### 4. 互动式教程 (Interactive Walkthrough)

```yaml
适用场景:
  - 复杂产品（需要手把手教核心流程）
  - 工具类产品（需要用户理解操作逻辑）

设计要点:
  ✅ 每次展示 1 个提示（不要让提示重叠）
  ✅ 提供"跳过教程"和"再看一次"选项
  ✅ 用高亮 + 箭头引导，而非全屏遮罩
  ❌ 不要超过 5 步（超过 5 步 = 重新设计产品）
  ❌ 不要每个按钮都加提示
```

## 多渠道引导矩阵

| 渠道 | 时机 | 内容 | 频率 |
|------|------|------|------|
| 📱 应用内提示 | 登录后即时 | 引导完成关键行为 | 第 1 次登录 |
| 📧 Day 1 邮件 | 注册后 1 小时 | 欢迎 + 快速入门指南 | 1 次 |
| 📧 Day 3 邮件 | 注册后 3 天 | 功能深度教程 | 1 次 |
| 📧 Day 7 邮件 | 注册后 7 天 | 案例/最佳实践 | 1 次 |
| 📧 Day 14 邮件 | 注册后 14 天 | 未激活用户专属优惠 | 1 次 |
| 💬 应用内消息 | 3/7/14 天登录时 | 个性化进度提醒 | 按需 |
| 📞 人工电话 | Day 1-3 | 大客户专属 Onboarding Call | 按需 |
| 📚 帮助中心 | 全时 | 自助服务 | 持续 |
| 🎥 视频教程 | 欢迎邮件 | "5 分钟上手"系列 | 持续更新 |

## AutoWash Pro X1 示例

### 示例 1：洗车机管理平台新用户引导

```yaml
产品: AutoWash Pro X1 云管理平台
用户画像: 洗车店/加油站经营者
技术背景: 非技术背景，需要极简引导

激活指标 (Aha Moment):
  "完成第一次远程启动 + 看到当日洗车统计报告"
  → 用户在 7 天内完成此行为 → Day 30 留存率提升 3x

引导路径:

  ⏱️ 第 0 天 — 注册完成 (即时):
    ┌────────────────────────────────────────┐
    │  🎉 欢迎加入 AutoWash Pro X1！          │
    │                                         │
    │  让我们在 5 分钟内完成设置:              │
    │                                         │
    │  ⬜ 1. 连接你的洗车机                    │
    │  ⬜ 2. 设置洗车套餐                      │
    │  ⬜ 3. 开启第一笔订单                    │
    │                                         │
    │  [开始设置]  [我有技术人员帮我]          │
    └────────────────────────────────────────┘

  ⏱️ 第 0 天 — 步骤 1: 设备连接:
    引导方式: 互动式教程
    Step 1: 扫描洗车机上的二维码
    Step 2: 确认设备名称（如 "入口洗车机"）
    Step 3: 看到绿色连接确认 → 🟢 连接成功！
    完成奖励: 🎊 "你的洗车机已上线！" 动画

  ⏱️ 第 0 天 — 步骤 2: 套餐设置:
    引导方式: 清单 + 空状态
    初始状态: "你还没有设置洗车套餐"
    ┌────────────────────────────────────────┐
    │  💡 我们准备了一些热门套餐:              │
    │  🟢 标准洗 $8  (最受欢迎)               │
    │  🔵 精洗 $15   (含打蜡)                 │
    │  🟣 至尊洗 $25 (含全套护理)             │
    │  ➕ 自定义套餐                           │
    └────────────────────────────────────────┘

  ⏱️ 第 0 天 — 步骤 3: 首单完成:
    引导方式: 模拟体验
    "试试手动启动一次洗车——就像客户来了一样"
    完成后弹窗: 当日统计面板预览

  ⏱️ 第 1 天 — 邮件:
    主题: "你的 AutoWash Pro X1 昨天表现如何？📊"
    内容:
      - 昨日洗车次数
      - 设备运行时长
      - 本周天气预报 → 预计客流量
      - "查看完整报告 →"

  ⏱️ 第 3 天 — 应用内:
    "你已经完成了 12 次洗车！看看怎么提高效率 👀"
    → 引导到「智能调度」功能

  ⏱️ 第 7 天 — 邮件:
    主题: "第一周总结 + 3 个进阶技巧 🔥"
    内容:
      - 本周洗车数据报告
      - 技巧 1: 设置会员卡 → 提升复购 30%
      - 技巧 2: 短信通知 → 减少客户等待
      - 技巧 3: 雨量感应 → 自动暂停，省水省电

  ⏱️ 第 14 天 — 未激活用户专属:
    条件: 7 天内没有完成 10 次洗车
    触发: "AutoWash Pro X1 还有这些功能你没用过……"
    内容:
      - 视频: 3 分钟的隐藏功能盘点
      - 客服: "需要一对一指导？预约时间 →"

多语言支持:
  - 中文: 中国大陆/台湾/东南亚华语区
  - 英文: 北美/欧洲/中东
  - 阿拉伯语: 中东/GCC 国家
  - 西班牙语: 拉丁美洲
  - 引导流程根据语言/地区自适应（如中东客户偏好 WhatsApp 通知而非邮件）
```

### 示例 2：空状态设计——设备未连接

```yaml
场景: 新用户登录后尚未连接任何洗车机

❌ 糟糕的设计:
  页面显示: "暂无设备"（空白页）
  用户感受: 😕 困惑，接下来该做什么？

✅ 优秀的设计:
  ┌──────────────────────────────────────────────┐
  │                                               │
  │          🚗💨  (洗车机动图插画)                  │
  │                                               │
  │      "连接你的第一台 AutoWash Pro X1"           │
  │                                               │
  │   📹 连接步骤（30 秒视频）                      │
  │   ┌─────────────────────────────────┐         │
  │   │ 1. 扫描机器二维码                  │         │
  │   │ 2. 连接 Wi-Fi                    │         │
  │   │ 3. 命名你的洗车机                  │         │
  │   └─────────────────────────────────┘         │
  │                                               │
  │      [🔗 连接洗车机]   [📖 查看说明书]          │
  │                                               │
  │  💡 提示: 连接后你将能看到实时洗车数据           │
  │  ┌─────────────────────────────────┐         │
  │  │ 📊 今日洗车: —    ⏱️ 平均时间: —    │         │
  │  │ 💰 今日收入: —    ⚡ 设备状态: —    │         │
  │  │   ↑ 连接后这里将显示实时数据       │         │
  │  └─────────────────────────────────┘         │
  └──────────────────────────────────────────────┘
```

---

> 🎯 **核心公式**: 激活率 × 留存率 × 推荐率 = 增长飞轮。激活是乘法中的第一个乘数——如果激活率为 0，后续一切归零。所以，找到你的 Aha Moment，然后让用户以最快的速度到达它。
